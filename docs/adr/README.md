# Architecture decision records

ADRs record durable decisions that affect multiple systems or constrain future work.

## Statuses

- **Proposed:** under review; not yet binding.
- **Accepted:** current decision.
- **Superseded:** replaced by a newer ADR that links back.
- **Rejected:** considered but not adopted.

## Required structure

Each ADR states context, decision, alternatives, consequences, evidence, owner, and review trigger.
Use a new sequential number. Do not rewrite accepted history; supersede it.
